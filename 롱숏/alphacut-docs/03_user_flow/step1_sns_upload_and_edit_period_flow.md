## FLOW A. SNS 직접 업로드 플로우

### A-0. 전제 조건

| 항목 | 내용 |
|------|------|
| 대상 플랫폼 | YouTube / Instagram / TikTok |
| 진입 시점 | 쇼츠 편집 완료 후 "업로드" 단계 |
| 연동 방식 | OAuth (각 플랫폼 계정 권한 위임) |
| 필요 권한 | YouTube: upload + yt.analytics.readonly / Instagram: instagram_content_publish / TikTok: video.publish |
| 관리 위치 | 서비스 내 '내 정보 관리' 페이지 (약관 근거) |

---

### A-1. 최초 플랫폼 연동 흐름

```
[편집 완료 쇼츠 화면 or 설정 페이지]
  └─> [업로드] 버튼 또는 [SNS 연동] 메뉴 클릭
        └─> 연동된 계정 없음 상태
              └─> 플랫폼 선택 UI 노출
                    ├─ [YouTube 연동하기]
                    ├─ [Instagram 연동하기]
                    └─ [TikTok 연동하기]

[YouTube 연동 선택 시]
  └─> Google OAuth 팝업 또는 리다이렉트
        └─> 권한 요청 화면:
              "AlphaCut이 다음 권한을 요청합니다"
              ✓ YouTube에 동영상 업로드
              ✓ YouTube 채널 데이터 보기 (분석)
        └─> [허용] 클릭
              └─> 알파컷으로 복귀 + 연동 완료 표시
                    └─> 연동된 YouTube 채널명 + 프로필 이미지 표시

[Instagram 연동 선택 시]
  └─> Meta OAuth 팝업
        └─> 권한 요청:
              ✓ Instagram 릴스(Reels) 게시
              ✓ Instagram 스토리(Stories) 게시
        └─> [허용]
              └─> 연동 완료 + Instagram 계정명 표시

[TikTok 연동 선택 시]
  └─> TikTok OAuth 팝업
        └─> 권한 요청:
              ✓ TikTok 동영상 게시
        └─> [허용]
              └─> 연동 완료 + TikTok 프로필 표시
```

**연동 상태 표시 UI (추정)**
```
┌─────────────────────────────────────────────┐
│  연결된 계정                                 │
│  ● YouTube   채널명 OOO          [연결 해제] │
│  ● Instagram @username           [연결 해제] │
│  ○ TikTok    연결되지 않음       [연결하기]  │
└─────────────────────────────────────────────┘
```

### A-2. YouTube 업로드 플로우 (고급 기능 아티클 기반)

```
[완성된 쇼츠 확인 페이지]
  └─> [업로드 정보 추천] 버튼 클릭 (고급 기능)
        └─> AI가 영상 분석 후 메타데이터 자동 생성:
              - 제목 (복사 버튼)
              - 설명글 (복사 버튼)
              - 태그/해시태그 (복사 버튼)
              - 카테고리 (복사 버튼)

[직접 업로드 선택 시 (추정)]
  └─> [YouTube에 업로드하기] 버튼 클릭
        └─> 업로드 설정 화면:
              - 제목 입력 (AI 추천값 기본 입력)
              - 설명글 입력
              - 태그 입력
              - 카테고리 선택
              - 공개 범위 선택: 공개 / 일부 공개 / 비공개 / 예약
        └─> [업로드 시작] 버튼
              └─> 업로드 진행 중 (프로그레스 표시)
                    └─> 완료 알림
                          └─> YouTube Studio 링크 또는 영상 URL 제공
```

---

### A-3. Instagram 업로드 플로우

```
[완성된 쇼츠 확인 페이지]
  └─> [Instagram에 업로드하기] 버튼 클릭
        └─> 업로드 설정 화면:
              - 업로드 유형 선택: [릴스(Reels)] / [스토리(Stories)]
              - 설명(캡션) 입력
              - 공개 범위 (Meta 정책 따름)
        └─> [업로드 시작]
              └─> 업로드 진행 중
                    └─> 완료 알림 + Instagram 게시물 링크

⚠️ 주의:
- AI로 생성/편집된 콘텐츠의 경우 Instagram 커뮤니티 가이드라인 준수 책임은 사용자에게 있음
- 저작권 침해 콘텐츠 업로드 금지 (약관 제5조)
```

### A-4. TikTok 업로드 플로우

  └─> [TikTok에 업로드하기] 버튼 클릭

              - 제목/설명 입력
              - 공개 범위 선택

### A-5. 연동 해제 플로우

```
[내 정보 관리 페이지]
  └─> 연결된 플랫폼 목록
        └─> [연결 해제] 버튼 클릭
              └─> 확인 다이얼로그:
                    "OOO 계정 연동을 해제하시겠습니까?
                     연동 해제 시 관련 데이터가 즉시 삭제됩니다."
              └─> [해제하기] 확인
                    └─> 연동 정보 삭제 완료
                          └─> 연동 목록에서 제거

또는:
  └─> 각 플랫폼 앱 설정에서 직접 권한 철회 가능
        - Google: 보안 설정 페이지
        - Instagram: Instagram 설정
        - TikTok: TikTok 설정
```

### A-6. 에러 케이스 (SNS 업로드)

| 에러 | 발생 조건 | 처리 |
|------|----------|------|
| 연동 만료 | OAuth 토큰 만료 | 재연동 유도 팝업 |
| 권한 부족 | 사용자가 일부 권한만 허용 | 필요 권한 안내 + 재인증 |
| 플랫폼 측 업로드 실패 | 플랫폼 서버 오류 / 용량 초과 | 오류 메시지 + 다운로드 후 직접 업로드 안내 |
| 커뮤니티 가이드라인 위반 | 플랫폼이 업로드 거부 | 플랫폼 정책 안내, 책임은 사용자에게 있음 |
| AI 콘텐츠 표시 미준수 (TikTok) | TikTok 정책 위반 감지 | TikTok AI 표시 정책 안내 |

## FLOW B. 편집 기간 제한 UX 플로우

### B-0. 정책 요약

| 구분 | 편집 가능 기간 |
|------|-------------|
| 비멤버십 (무료 30분 이용권 포함) | 최초 생성일로부터 **10일** |
| 멤버십 가입자 (Basic/Pro/Max) | 최초 생성일로부터 **1개월 (30일)** |

근거: 이용약관 제4조 4항

---

### B-1. 프로젝트 목록 — 기간 상태 표시 UI

```
┌──────────────────────────────────────────────┐
│  쇼츠 썸네일                                  │
│  제목: AI가 생성한 쇼츠 제목                  │
│                                              │
│  [✅ 편집 가능]   D-7 남음                   │  ← 만료 7일 전부터 표시
│  또는                                        │
│  [⚠️ 편집 마감 임박]  D-3 남음               │  ← 3일 전 강조 경고
│  또는                                        │
│  [🔒 편집 기간 만료]  2026-04-10 만료         │  ← 만료 후 잠금 상태
│                                              │
│  [편집하기]  [다운로드]  [재생성]             │  ← 만료 시 편집 버튼 비활성화
└──────────────────────────────────────────────┘
```

---

### B-2. 편집 기간 만료 전 알림 플로우

```
[편집 기간 만료 D-3 도달 시]
  └─> (추정) 서비스 내 알림 또는 이메일 발송
        └─> 알림 내용:
              "OO 쇼츠의 편집 기간이 3일 후 만료됩니다.
               만료 전에 편집을 완료하거나 다운로드하세요."
        └─> CTA:
              [지금 편집하기] → 편집 화면 이동
              [다운로드하기] → MP4 다운로드

[편집 화면 내 배너 표시]
  └─> 상단 또는 우측 패널에 기간 만료 경고 배너:
        "⚠️ 편집 가능 기간: 앞으로 3일 남았습니다"
        [멤버십 업그레이드로 기간 연장하기] → /pricing
```

---

### B-3. 편집 기간 만료 시 접근 차단 플로우

```
[프로젝트 목록에서 만료된 쇼츠 [편집하기] 클릭]
  └─> 편집 화면 진입 차단
        └─> 인라인 메시지 또는 모달:
              "편집 기간이 만료되었습니다.
               최초 생성일로부터 [10일 / 1개월]이 경과하여 편집이 제한됩니다."
              └─> CTA 분기:
                    ├─ 비멤버십 → "멤버십 가입 시 1개월로 편집 기간이 연장됩니다"
                    │             [멤버십 시작하기] → /pricing
                    └─ 멤버십 → "다운로드 후 재업로드 또는 재생성을 이용해보세요"
                                [재생성하기] → 이용권 50% 차감으로 새 쇼츠 생성
                                [다운로드하기] → 완성 파일 다운로드 (편집 없이)

[다운로드는 만료 후에도 가능]
  └─> 다운로드 버튼은 기간 무관하게 활성 유지 (추정)
```

---

### B-4. 멤버십 전환으로 기간 연장 플로우

```
[만료 경고 CTA 클릭] → /pricing 이동
  └─> 요금제 선택 (Basic/Pro/Max)
        └─> 결제 완료
              └─> 멤버십 활성화
                    └─> 기존 쇼츠 편집 기간 자동 갱신 여부 (확인 필요 항목 ⚠️)
                          ├─ 시나리오 A: 가입 시점부터 1개월로 리셋
                          └─ 시나리오 B: 이미 만료된 쇼츠는 복구 불가
```

### B-5. 에러 케이스 (편집 기간)

| 에러 | 발생 조건 | 처리 |
|------|----------|------|
| 편집 기간 만료 | D+10 또는 D+30 경과 | 편집 버튼 비활성 + 만료 배지 |
| 만료 직전 저장 실패 | 만료 당일 편집 중 | 저장 완료 후 알림, 기간 종료 안내 |
| 멤버십 해지 후 기간 축소 | 멤버십 → 무료로 변경 | 기간 재계산 정책 확인 필요 ⚠️ |
