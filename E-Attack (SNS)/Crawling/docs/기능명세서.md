# SNS 트렌드 분석 및 AI 재구성 자동 배포 — 기능명세서

## 1. 프로젝트 목표
특정 커뮤니티(아이보스, 스레드, X)의 인기 게시물을 크롤링하여 UI에 보여주고,
사용자가 선택한 게시물을 AI가 분석/재구성하여 다양한 SNS 채널에 맞춤형으로 자동 게시하는 웹 애플리케이션 구축.

---

## 2. 상세 기술 요구사항

### [Step 1-3: 크롤링 및 메인 UI]
- **Target URL:**
  - 아이보스: https://www.i-boss.co.kr/ab-2108 (마케팅 양식 게시판)
  - 스레드/X: 실시간 트렌드 또는 특정 키워드 검색 결과
- **기능:** Playwright 라이브러리를 사용해 상위 10개 게시물의 [순번, 제목, 본문 원문, 이미지 URL, 조회수, 좋아요, 댓글, 공유수]를 크롤링하여 테이블 형태로 렌더링.
- **UI:** '새로고침' 버튼 클릭 시 크롤링 시작. 행 클릭 시 하단에 '해당 레퍼런스로 재구성하기' 버튼 노출.

### [Step 4: AI 추천 및 키워드 입력]
- **AI 역할:** 선택된 게시글의 `content_raw`를 분석하여 연관된 '추천 키워드 3개' 생성.
- **형식:** 스톡 이미지가 배경으로 깔린 모달창에 추천 키워드와 사용자의 주관적 입력창 배치.

### [Step 5-6: 플랫폼 및 계정 선택]
- **대상:** 스레드, 블로그(네이버/티스토리), X, 아이보스, 링크드인.
- **로직:** 미리 저장된 계정 정보를 선택할 수 있는 드롭다운 메뉴 구현.

### [Step 7: AI 콘텐츠 재구성 (템플릿)]
- **프롬프트:** "선택된 원문의 구조를 유지하되, 사용자가 입력한 키워드를 반영해 [플랫폼 규격]에 맞춰 새로 작성하라."
- **규격:**
  - 스레드: 500자 내외, 구어체, 이모지 적극 활용.
  - X: 280자 내외, 강렬한 훅(Hook) 문장 포함.
  - 블로그: 장문의 정보 전달형, 서론-본론-결론 구조.

### [Step 8: 최종 편집 및 배포]
- **편집기:** AI가 쓴 글을 관리자가 최종 수정할 수 있는 Text Area 제공.
- **배포:** '배포' 버튼 클릭 시, 해당 플랫폼의 API를 호출하거나 자동화 브라우저(Playwright)를 통해 실제 게시물 업로드 시뮬레이션 수행.

## 3. 기능 상세

### 기능 1: 인기글 기반 트렌드 분석
> 중요도: 높음 | 상태: 시작전

**수용 기준:**
1. 인기글 수집 — 외부 커뮤니티의 인기 게시물을 주기적으로 수집하고 DB에 저장
   - 1.1. 스케줄링 설정 — 주기적으로 크롤링 작업을 실행하는 스케줄러 설정
   - 1.2. 외부글 크롤링 — 지정된 커뮤니티의 인기 게시물 목록과 본문 데이터 수집
   - 1.3. 데이터 정제/저장 — 수집한 데이터를 정제하여 분석에 용이한 형태로 DB에 저장
   - 1.4. 수집 실패 처리 — 크롤링 대상 사이트 접속 실패 또는 HTML 구조 변경 시 에러 로그 기록 및 알림

2. 트렌드 대시보드 — 수집된 인기 게시물을 시각적으로 제공하고 상호작용
   - 2.1. 인기글 목록 API — 저장된 인기 게시물 목록을 필터링/정렬하여 제공
   - 2.2. 인기글 UI 렌더링 — API로 받은 인기글 목록을 카드 형태로 보여주는 UI
   - 2.3. 게시물 상세 조회 — 사용자가 특정 게시물을 선택하면 원문 링크와 요약 내용을 모달에 표시
   - 2.4. API 요청 실패 — 인기글 목록 API 호출 실패 시 에러 메시지 표시 및 재시도 옵션

### 기능 2: 프로젝트 및 채널 관리
> 중요도: 높음 | 상태: 시작전

**수용 기준:**
1. 프로젝트 생성/관리 — 클라이언트별 프로젝트 CRUD
   - 1.1. 프로젝트 목록 API
   - 1.2. 프로젝트 생성 UI
   - 1.3. 프로젝트 정보 저장
   - 1.4. DB 저장 실패 처리

2. 채널/톤앤매너 설정 — 프로젝트별 소셜 채널 연동 + 톤앤매너 설정
   - 2.1. 채널 연동 API (OAuth)
   - 2.2. 톤앤매너 설정 UI
   - 2.3. 설정값 DB 저장
   - 2.4. 채널 연동 실패 처리

### 기능 3: AI 레퍼런스 기반 콘텐츠 생성
> 중요도: 높음 | 상태: 시작전
